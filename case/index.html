事例「AzureとCodeを駆使して価値を生み出す」

2018.3.31

  • share

Microsoft Azureの実績に関しては150事例超、基幹系、EC系、金融系、大規模SNS等とAzure上初事例というものも含めて業界をリードしています。

弊社はAzure専業企業として、2008年の黎明期より取り組んでいます。なぜAzureを活用するかはこちらに解説しています。

事例に関しては、大きく以下のジャンルにわかれますが、 継続的に改善が可能で、変化に強く、一度構築した価値が顧客や業界に残るような形で提供していくことを目指しています。 この原動力となっているのがAzureの各機能であり、それを最大限に活かす我々の技術力です。

Eコマース

　

EC業界は日々加速度的に成長しているとともに、継続的な変化に対応する必要があります。Commerbleはこれらの課題を解決すべく、クラウドネイティブなサービスとして構築されています。この機能拡張や連携にもAzureの機能を活用します。

建築資材のEコマースサイト。

ECおよびコーポレートサイトを店舗とEC共通の会員基盤で。ラインビジネスコネクトの活用でチラシ費用の削減など。

アパレル、化粧品ほか

PCI DSS

　

長期間のプロジェクトかつ高コストになりがちな PCI DSS への準拠を、Microsoft Azure の PaaS (Platform as a Service)、および、 世界的に活用されている「 Azure のセキュリティとコンプライアンスのブループリント」を活用することによって、 迅速かつ適切なコストで実現するシステム構築サービスを提供します。 PaaS を最大限に活用することにより、自前のデータセンターや IaaS (Infrastructure as a Service) を利用した場合に、ユーザ自身で PCI DSS 対応の責務を負っていた部分について、Microsoft へ PCI DSS 対応を移譲することができます。

クレジットカード会員サービス(PCI DSS)

PCI DSS に準拠したクレジットカード会員向けサービスをAzure上に構築。Azure のPaaSのみで PCI DSS に準拠。 これまで大手SI企業が高コストで構築していたシステムをAzureを活用して手早く、適切なコストで構築。
大規模ポイントシステム基盤

推定利用ユーザー2000万人のオンラインサービスの基盤機能を構築。AzureのPaaS機能を先方のセキュリティ条件に適用する形で対応。PaaSを利用し、インフラの運用管理をクラウド側に任せる形で、迅速かつセキュアなインフラ基盤を作成した。

CRM

　

店舗会員とEC会員のシームレスな統合、双方向への導線や誘導は、仕組みやデザインなどやるべきことが多岐にわたります。 またコミュニケーションチャネルとして、Line(一斉配信だったり、Business Connectでの個別識別) 、大規模なマーケティングメール、会員証アプリ、ソーシャル(Facebook、Twitter)、メール配信、DMなど顧客の性別や属性、事業形態に応じた対応を行います。 AzureのPaaSを活用し、組み合わせ柔軟に構築したものを、顧客のニーズに応じたチャネルで提供します。

大手CRM企業Azure基盤

CRM大手企業の事業基盤をAzure上に構築。コンサルティングと部分的な実装支援。
店舗CRM基盤(複数事例)

全国に100店舗以上ある店舗のCRM基盤を構築。O2O対応として、オンラインの会員機能を設置。基幹側と連携し、顧客管理および、特定の条件の顧客に対し、DMを送る基盤を構築。SendGridのマーケティングメール機能と連携しており、多機能なDM基盤となっている。
ラインビジネスコネクトSaaS

ラインビジネスコネクトの大規模配信機能を備えたSaaSをAzure上に構築。各企業において個別に実装するのではなく、SaaSプラットフォームとして提供するサービスを構築。その大規模配信部分(キューイングやリトライの仕組みを含む)をAzure上の機能を用いて効果的に実装した。
ヘルスケア企業SNSサイト

Facebookと連携した健康相談サイトを作成。会員管理機能やQAの仕組みを作成。
自動車会社系CMS+SNSサイト

CMS機能としてユーザーがコミュニティを作り、独自に情報を発信できるサイトとした。その際に利用するSNSアカウントでは主要なWEBサービスと連携して、利用できるようにした。Facebook、Twitter、Google+、Instagram、Tumblrに対応。アンケート機能も装備。ユーザー主導型のCMS機能を兼ねたサイトとして実証実験を行った。

CMS

コンテンツ管理の最適な粒度を設計、継続的なデザイン改善、負荷対策、ワークフローなどの運用管理の実現、バージョニング、マーケティング機能との連携など考慮する点が多岐にわたります。 課題要件に対しAzureの機能を最適に使うことや、サイトコアやCommerbleのような設計自由度の高い CMSを駆使することによって実現します。

外資系ソフトウェア企業、パートナー管理サイト

パートナーとの関係に応じた情報発信を可能とした。会員の管理や、その閲覧権限をサイトコアの機能とうまく組み合わせて対応している。 他に、ドキュメント検索エンジンを別途構築した。
LINE連動キャンペーンサイト

LINE連動時にわずか30分の間に50万人のユーザーが集まる負荷にも耐える。キャンペーンと連動する会員機能を併設。各ユーザーの行動や嗜好をトラッキングし、レコメンドの表示も行っている。
大手製造業グループ企業サイト(複数事例)

グループ会社のサイトが乱立していたのに対し統合基盤としてサイトコアCMSの環境を提供。エンタープライズ企業が求める各種要件に対応。また運用等を改善するため、Azureのマネージドサービス(WebApps,SQL Database)を中心に組み立てた。
公共系サイト(複数)

大規模公共系のサイトを複数。季節的なトラフィック変動があるサイトを安定稼働させる。
複数の企業コーポレートサイト

複数の企業コーポレートサイト、デジタルマーケティング基盤を実装。
大手新聞社フォトサイト

大規模なアクセスに対応。月間のPVが1000万前後で震災時には海外からのトラフィックが激増し2000万PVを超えた。

基幹、業務系システム(業務基幹、通販基幹)

　

基幹系の業務は、ビジネスモデルと今後の変化を十分に考慮した上で、業務フローを丁寧に検討することから始めます。管理画面を必要とする業務、実際は人での不要な自動化可能な業務、切り捨てるべき業務など、システムに携わる者として継続的な改善可能な課題が多く醍醐味としてあります。これらをAzure のPaaSを中心とした機能で適切に組み上げます。

倉庫業基幹システムPMO

各種、交通整理を行い、プロジェクトが円滑に進むよう支援。システムアーキテクチャ、コミュニケーションの整理。各フェーズにおける課題の解決。
ブライダル企業基幹システム

大手ブライダル企業の基幹システムをAzure上に構築。全世界の拠点別データベースをAzureのPaaS上に移行した。大規模利用におけるSQLServer2000からSQL Databaseの移行を実現した。
大規模ECの基幹システム(複数事例)

モールを含めた注文管理、商品管理、複数倉庫の連携やPOS連携も含む通販基幹システムを構築。
幼稚園向け管理サービスの提供

マルチテナント型の幼稚園向け管理パッケージをAzure上に構築。生徒の情報や請求回収機能、課外授業の受講、各種レポートなど、幅広い機能に対応したパッケージを構築した。
工場管理FAQシステム

工場で利用される多言語QAシステム、課題管理サービスを構築。工場やラインに紐づくユーザーの管理。ラインからの問い合わせに対する回答、事例など類似ケースの情報公開、など各種特殊な業務を実現している。
大手企業向けストレージサービスの拡張

自社クラウドにオブジェクトストレージとして、Azureストレージを機能として追加できるように管理プラットフォームを構築。
Azure関連ドキュメント(複数)

自学自習ドキュメントが整備されていない中で、必要なシナリオを精査し、Microsoft社向けの自習書を執筆した。当時の日本の最先端の知見がまとまっている。

大規模データ解析

　

大規模データ解析は、適切な粒度での設計、運用時のリカバリー、拡張性、負荷の流動性などをより考慮する必要があります。Azure Functions などサーバーレス、PaaSと呼ばれるマネージドなものでの連携を構築し、変化に強い連携を常に目指しています。

プライベートDMP

プライベートDMPとしてユーザートラッキング、アクセス履歴の取得、SNS情報と連携した個人情報の取得、嗜好情報の分析、レポート機能 をAzure上の機能を用いて効果的に実装した。月間のアクセス規模は数千万PV規模で安定したパフォーマンスを提供。
大手ニュースサイト全文検索機能

大手ニュースサイトの全文記事検索サービス。ソーシャルでの反応を集計する事にも対応している。全文検索にLucene.NETを利用。
Azure Search導入支援

公共系のデータを解析するAzure Search基盤の導入支援。
大規模ソーシャルゲーム＋キャンペーンサイト(複数事例)

ストレージの性能を最大限に活かすためにフルスクラッチで実装した。大きなトラブルなく200万人のユーザーのトラフィックに対応した。インスタンスは最大で70台程度を活用。Facebookアプリに対応。

アプリ(iOS Android=>Xamarin)

　

アプリは継続的なアップデートやOSの仕様変更に追従する必要があります。Xamarinでi OSとAndroidに対応するクロスプラットフォームなアプリを構築、 Swagger でAPI集約された効率的な開発を進めています。

すこやこアプリ

母子手帳や病歴の管理など子育てを支援するアプリ。API側を中心に支援。ベビカム社提供。
FEED

#IoT

　

現在は生体ロガーの事業(Biologging Solutions)に未来を感じ、支援、協力させていただいています。 同社の哲学(地球のデータを未来のためにストアする、研究プラットフォームを提供する)という理念は素晴らしく、世の中に広く普及することを願っています。

生体ロガー情報解析サービス

スタートアップ企業の新規サービス構築。生体情報を取得するロガーのデータを受け付け、マシンラーニングなどで分析するサービスの作成。
